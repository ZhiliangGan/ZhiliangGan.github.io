2. 迁移原则#

2.1 三不原则#

  1. 不删除代码：所有待废弃代码移入 legacy/ 目录，保留历史
  2. 不中断开发：迁移期间现有流水线照常运行
  3. 不破坏接口：SAF 接口先稳定再对接

2.2 迁移顺序#

Phase 1: 稳固 SAF 内核（共享层先完成）
Phase 2: 废弃 StrategeFactoryFlow 中对应模块
Phase 3: 废弃 TSS 中对应模块
Phase 4: 清理 legacy/ 目录
